# Test Scenarios — readme-generator

Each scenario defines the input condition, expected behaviour, and pass/fail criteria.

---

## T01 — Skill activates on correct trigger phrases

**Input**: User says "write me a README for this project"
**Expected**: Skill loads and reads repo files before writing anything
**Pass criteria**: At least one manifest file (`requirements.txt`, `package.json`,
  etc.) is read before any markdown content is generated

---

## T02 — Skill does NOT activate on architecture requests

**Input**: User says "generate an architecture document"
**Expected**: `architecture-audit` skill activates instead
**Pass criteria**: `readme-generator` skill is not loaded

---

## T03 — Existing README triggers confirmation prompt

**Input**: Repo already has a `README.md`; user asks to "generate a README"
**Expected**: Skill asks whether to overwrite or use a different filename
**Pass criteria**: File is not written until user confirms

---

## T04 — No content invented without evidence

**Input**: Repo has no `.env` file and no `os.getenv` calls in source
**Expected**: Environment variables section is omitted
**Pass criteria**: No "VARIABLE_NAME" placeholder rows appear in output

---

## T05 — Running tests section omitted when no tests exist

**Input**: Repo has no test directory, no pytest.ini, no jest.config
**Expected**: "Running tests" section is absent from output
**Pass criteria**: Output contains no "Running tests" heading

---

## T06 — License section omitted when no LICENSE file exists

**Input**: Repo has no `LICENSE` or `LICENSE.md` file
**Expected**: "License" section is absent from output
**Pass criteria**: Output contains no "License" heading

---

## T07 — Secret values are redacted

**Input**: Repo has `.env` with `REDIS_PASSWORD=abc123`
**Expected**: README shows `<your-value>` not `abc123`
**Pass criteria**: No literal secret value appears in the generated file

---

## T08 — Output stays under 150 lines

**Input**: Any valid trigger phrase on a typical repo
**Expected**: Generated README is under 150 lines
**Pass criteria**: `wc -l README.md` returns < 150

---

## T09 — No architecture or security content

**Input**: Any valid trigger phrase
**Expected**: Output contains no Mermaid diagrams, security findings,
  or performance assessments
**Pass criteria**: Terms like "CRITICAL", "MEDIUM", "N+1", "CVE",
  or Mermaid code fences are absent from output

---

## T10 — Author credit is present

**Input**: Any valid trigger phrase
**Expected**: Last line of output is `Generated by @simplymanas`
**Pass criteria**: Credit is present and correctly spelled
