# Section Checklist — readme-generator

Run through this checklist before finalising the generated README.
Every item marked ❌ or ⚠️ must be resolved before writing the file.

---

## Content checklist

| # | Section | Check | Status |
|---|---|---|---|
| 1 | Project name | Taken from `package.json`, `pyproject.toml`, or repo folder — not invented | |
| 2 | Description | One sentence, no marketing language, no buzzwords | |
| 3 | Tech stack | Populated from actual dependency manifests, not assumed | |
| 4 | Prerequisites | Lists only what the operator installs manually | |
| 5 | Installation | Commands verified against actual manifest files | |
| 6 | Env vars | Extracted from `.env*`, config source, or Dockerfile — not guessed | |
| 7 | Env vars | All secret values replaced with `<your-value>` placeholders | |
| 8 | Running locally | Command verified against Dockerfile CMD, Makefile, or npm scripts | |
| 9 | Running tests | Section present only if test runner config exists in repo | |
| 10 | Project structure | Top-level dirs only, one line each | |
| 11 | License | Section present only if LICENSE file exists in repo | |

---

## Quality checklist

| Check | Status |
|---|---|
| Total length is under 150 lines | |
| No architecture diagrams included | |
| No security analysis included | |
