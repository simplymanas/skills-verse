# README Generator

## Purpose

Produce a single, concise `README.md` that lets a developer clone, install,
configure, and run the project in under 5 minutes. This is a quick-start
document, not full documentation.

## Procedure

1. **Inspect the repo before writing anything.** Read:
   - Package/dependency manifests (`package.json`, `requirements.txt`,
     `pom.xml`, `go.mod`, `*.csproj`, etc.)
   - Config files and `.env.example` / env-var references in code
   - Dockerfiles / docker-compose files
   - Test configuration (test runner, scripts)
   - Any existing CI config (only to confirm test/build commands, not to
     document the pipeline itself)

2. **Ground every claim in what you find.** Do not invent commands, env
   vars, or steps. If something can't be verified from the repo, omit it —
   do not guess or assume.

3. **Write `README.md` with exactly these sections, in this order:**
   1. Project name + one-sentence description (what it does, no marketing language)
   2. Tech stack (language, framework, key libraries — only what's actually used)
   3. Prerequisites (runtime versions, required tools)
   4. Installation (exact commands, taken from actual package manager files)
   5. Environment variables (name, purpose, required/optional — pulled from
      actual `.env.example` / config code)
   6. Running locally (exact command(s) to start the dev server/app)
   7. Running tests (exact command — only if a test setup exists in the repo)
   8. Project structure (top-level directories only, one line each — not a full tree)
   9. License (only if a LICENSE file exists)

## Constraints

- Total length: aim for under 150 lines.
- No architecture diagrams, no security analysis, no performance review —
  those belong to the architecture-audit skill, not this one.
- No marketing language. No "Contributing" / "Roadmap" sections unless the
  repo already shows evidence of a contribution process (e.g. CONTRIBUTING.md).
- Every command must be copy-pasteable and verified against actual config files.
- Single output file: `README.md`.
- If a README.md already exists, ask the user whether to replace it or write
  to a different filename before overwriting.
