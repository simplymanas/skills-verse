# Example Trigger Phrases — first-principles-audit

These are real user messages that should activate this skill.

## Direct requests
- "Audit my codebase like you're the new CTO on day one"
- "Do a first-principles audit of this repo"
- "What would a fresh set of eyes kill in this codebase?"
- "What is everyone too close to see?"
- "Give me a contrarian take on this architecture"
- "What should we kill that people are afraid to touch?"
- "Do an opinionated teardown of the whole codebase"
- "Audit my codebase and database — no sacred cows"

## Implicit requests
- "We've been heads-down for two years — what are we missing?"
- "New CTO joining next month — what should they be told on day one?"
- "Be brutal about what's wrong with this system"
- "What's the one bet you'd make to change this architecture in 90 days?"
- "Give me your honest take, not the polished version"

## Boundary cases — should NOT trigger this skill
- "Write an architecture document" → use `architecture-audit`
- "Generate a README" → use `readme-generator`
- "Review this PR" → use `code-review`
- "Explain how this works" → use `explain-code`
